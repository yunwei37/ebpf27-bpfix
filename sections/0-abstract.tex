\begin{abstract}
eBPF lets developers run custom programs inside the Linux kernel, where a static verifier must prove each program safe before loading.
% eBPF 让开发者在内核里运行自定义程序，而每个程序加载前都必须由静态 verifier 证明安全。
The verifier proves safety one instruction at a time, building the proof that each value is used safely, such as a pointer staying within bounds.
% verifier 逐条指令地证明安全，为每个值建立"它被安全使用"的 proof，例如某个指针始终在界内。
When a program is rejected, the verifier returns a log of the per-instruction abstract state, ending in a one-line terminal error that marks where verification stopped.
% 当程序被拒时，verifier 返回一份记录每条指令处抽象状态的 log，末尾是一行 terminal error，标出 verification 停下来的位置。
An empirical study of 235 reproduced rejections shows the error cannot identify where the program lost the required proof, as one error covers as many as nine distinct root causes.
% 一项对 235 个复现 rejection 的实证研究表明，这行 error 判断不出程序在哪丢失了所需的 proof，因为一条 error 最多跨 9 个不同的 root cause。
Existing tools map the error to a source line, leaving the lost proof for the developer to recover.
% 现有工具只是把这行错映到一个源码行，丢失的 proof 仍要开发者自己去找。
We present \sys, which reads the same log and reconstructs, from the per-instruction states, where the required proof was established and where it was lost.
% 我们提出 \sys：它读同一份 log，从逐指令的状态重建出所需 proof 在哪建立、在哪丢失。
\sys localizes that loss, giving the developer the place to fix.
% \sys 定位到这个丢失点，给出开发者该改的地方。
On 75 repair tasks, replacing the raw verifier log with the \sys localization raises one-shot repair across three downstream models, from 22 to 38 of 75 on the strongest, with each fix accepted only by an oracle independent of \sys.
% 在 75 个修复任务上，把 raw verifier log 换成 \sys 的定位结果，在三个下游模型上都提升了 one-shot 修复，最强模型从 75 例中的 22 提到 38，而每个 fix 都只由一个独立于 \sys 的 oracle 接受才算数。
\end{abstract}
